将Wordpress接入邮件发信平台

2026-04-14 · 1 分钟阅读
邮件 发信 邮箱 博客 Wordpress

将 WordPress 接入发件平台，能方便地通过邮件验证的方式开放注册账号，让你的博客看起来更加专业；同时，当博客被评论、有新用户注册时，也能及时通过邮件发送到你的邮箱通知你。

与营销邮件不同，这种邮件一般称作事务型邮件。事务型邮件是由用户行为自动触发、以一对一形式发送的功能性邮件，通常这种邮件被当作垃圾邮件的可能性更低，被拒收的次数也更少。

对于轻度使用通知邮件的站点，这里推荐使用 Brevo 邮件发送平台。

Brevo 平台给每位用户提供每天 300 封的发信额度，对于中小型网站来说，通常已经足够使用了。
如果网站有较大量的发信需求，这里也推荐 Zoho Zepto 平台和 Amazon SES，通常性价比会更高一些。

这里以 Brevo 平台做演示。

首先注册好一个 Brevo 平台账号。
如果 Brevo 注册过程中出现因网络原因导致的问题，或者验证相关问题，本文不提供相应方案，请自行解决。

注册完成后，首先我们需要绑定一个自己的发信邮箱域名。点击右上角的齿轮 Settings 按钮，接着选择 Senders, domains, IPs 模块。

选择 Domains，添加好一个域名。接着在你的域名 DNS 平台按照要求设置好对应的解析记录。

等解析生效后，再选择 Senders，添加你想要显示的邮件发件人名称和邮件地址。

注意：发信邮件地址的域名，必须是你已经在 Domains 里面添加好并且解析生效的域名。

然后选择 SMTP & API 模块，再进入 API keys & MCP，生成一个 API，并妥善保存。

接着我们打开 WordPress 博客的管理页面，安装并启用 Brevo 插件。

然后点击左侧的 Brevo 板块，将刚刚生成的 API 粘贴到对应的位置。

接着，我们打开“事务性电子邮件”按钮，这样就算成功配置好了。

完成这些设置后，我们就可以开放注册功能了，用户也可以通过电子邮件来完成注册验证。这样一来，不仅博客的注册流程会更规范，在博客收到评论、有新用户注册时，你也能及时收到邮件通知。
